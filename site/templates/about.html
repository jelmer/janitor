About the Janitor

What is the Janitor?

The Janitor is an automated system that makes changes to version control repositories. It can automatically fix common issues, apply systematic improvements, and propose changes through merge proposals on various hosting platforms.

How It Works

  1. Detection: The Janitor identifies repositories that could benefit from automated fixes
  2. Execution: It runs specialized tools and scripts to make improvements
  3. Building: Changes are tested by building the modified code
  4. Proposal: Successful changes are proposed as merge requests

Supported Platforms

  • GitHub (Pull Requests)
  • GitLab (Merge Requests)
  • Launchpad (Merge Proposals)
  • Direct Git pushes (with appropriate permissions)

Common Fixes

The Janitor can handle various types of improvements:

  • Lintian fixes for Debian packages
  • New upstream version updates
  • Build dependency updates
  • Standards version bumps
  • Homepage URL updates
  • Repository relocation tracking
  • And many more...
