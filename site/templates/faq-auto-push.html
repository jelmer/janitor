This is great. How do I get it to automatically push improvements to my repository?

Simply give the bot commit access to your repository, and it will push fixes rather than proposing them.

The bot will need permission to push to the relevant branches. For repositories on a GitLab instances (such as salsa) this means that it will need developer permissions if the relevant branch is unprotected, and maintainer permissions if the relevant branch is protected. See the GitLab permissions guide for details.

For GitHub repositories, the bot needs:

  • Write access to the repository
  • Permission to push to the target branch (may require special configuration for protected branches)

Note: Direct pushes will only happen for changes that meet certain quality thresholds. More experimental or risky changes will still be submitted as merge proposals even if the bot has push access.
